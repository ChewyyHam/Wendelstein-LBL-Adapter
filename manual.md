# 1. Introduction

This repository provides a lightweight adapter and analysis toolkit for applying the Line-by-Line (LBL) radial-velocity pipeline to reduced spectra obtained with the Wendelstein Observatory FOCES spectrograph.

The repository consists of two independent components:

- **`wendelstein_lbl_adapter.py`**, which converts reduced Wendelstein spectra into FITS files compatible with the LBL Generic Instrument.
- **`lbl_analysis.py`**, which provides interactive visualization and quick-look analysis of the radial-velocity products generated by LBL.

This repository is **not** a replacement for the official LBL package. Instead, it extends the LBL framework by providing the additional tools required to process Wendelstein/FOCES data. Users should install the official LBL package separately and follow the official LBL documentation for the general installation procedure.

The intended workflow is

```text
Reduced Wendelstein spectra
            │
            ▼
wendelstein_lbl_adapter.py
            │
            ▼
LBL Generic Instrument
            │
            ▼
LBL radial-velocity products
            │
            ▼
lbl_analysis.py
```

The purpose of this manual is to document the Wendelstein-specific workflow, configuration, and implementation details that are not covered by the official LBL documentation.

## `wendelstein_lbl_adapter.py`

This script converts reduced Wendelstein/FOCES spectra into FITS files compatible with the LBL Generic Instrument.

Its main responsibilities include

- reading reduced Wendelstein spectra;
- selecting the desired spectral orders;
- generating science and blaze FITS files for LBL.

This script should be executed before running LBL.

---

## `lbl_analysis.py`

This script provides interactive visualization of the radial-velocity products generated by LBL.

Currently supported analysis includes

- relative radial-velocity time series;
- Lomb–Scargle periodograms;
- phase-folded radial-velocity curves.

The script accepts either a single `.rdb` file or a directory containing multiple `.rdb` files.

# 4. Data Conversion Workflow

The recommended workflow for processing Wendelstein/FOCES data with LBL is shown below.

```text
LBL installation and setup
            │
            ▼
Create the LBL directory structure
            │
            ▼
Convert Wendelstein spectra
            │
            ▼
Generate science and blaze FITS files
            │
            ▼
Modify the wrap file
            │
            ▼
Run LBL
```

Following the official LBL setup before running the adapter is recommended. This allows the generated science and blaze FITS files to be written directly into the corresponding LBL input directories, eliminating the need to move files manually later.

Within this workflow, the adapter performs three main tasks:

```text
Reduced Wendelstein spectra
            │
            ▼
Read FITS files
            │
            ▼
Select spectral orders
            │
            ▼
Generate science and blaze FITS files
```

The following sections briefly describe each step of the conversion process.

---

## 4.1 Read FITS Files

The adapter reads reduced Wendelstein/FOCES spectra from the specified input directory.

Each observation is processed individually and prepared for conversion into the format required by the LBL Generic Instrument.

---

## 4.2 Select Spectral Orders

The adapter allows the user to specify the lower and upper physical orders to be included in the LBL analysis.

Both limits are **inclusive**.

For example,

```text
Lower order : 84
Upper order : 114
```

selects physical orders **84–114**.

Only the selected orders are written to the output FITS files.

---

## 4.3 Generate LBL-Compatible FITS Files

For each observation, the adapter generates

- one science FITS file;
- one blaze FITS file.

These files follow the format required by the LBL Generic Instrument and can be used directly as the input for LBL.


# 5. Running LBL

After the science and blaze FITS files have been generated, the remaining steps follow the standard LBL workflow.

## 5.1 Install and Set Up LBL

Install the official LBL package and complete the initial setup by following the official LBL documentation.

During the setup process, LBL creates the required directory structure and generates a default wrap file for your project.

---

## 5.2 Configure the Wrap File

Before running LBL, the generated wrap file must be modified for Wendelstein/FOCES data.

Use the provided `sample_wrap.py` as a reference and update the corresponding parameters in your automatically generated wrap file.

The required Wendelstein-specific configuration options are described in **Chapter 6**.

---

## 5.3 Run LBL

Once the wrap file has been configured, execute it using Python:

```bash
python your_wrap_file.py
```

LBL will process the converted science and blaze FITS files and generate the radial-velocity products for further analysis.

The resulting `.rdb` files can then be analyzed using `lbl_analysis.py`.


## 6.1 Generic Instrument Parameters

Since Wendelstein/FOCES is not natively supported by LBL, this repository converts the reduced spectra into the format expected by the **Generic Instrument** interface. Consequently, the generated wrap file should be configured to use the Generic Instrument throughout the LBL workflow.

The default wrap file generated by LBL contains placeholder values (`None`) for several Generic Instrument parameters. Before running LBL, these parameters should be replaced with values appropriate for the target being analyzed.

### Instrument Selection

The instrument should be set to

```python
rparams['INSTRUMENT'] = ['Generic']
```

This instructs LBL to use the Generic Instrument interface when reading the converted science and blaze FITS files.

### Object Comparison

`OBJECT_COMPARISON` specifies the object that will be used to construct the template spectrum.

If no template is provided, LBL will automatically build one from all observations of the specified target. Therefore, this parameter should normally be set to the name of the science target.

For example,

```python
rparams['OBJECT_COMPARISON'] = ['TOI5786']
```

where `TOI5786` is the target being analyzed.

### Object Effective Temperature

LBL also requires an estimate of the target's effective temperature for template generation.

For example,

```python
rparams['OBJECT_TEFF'] = [5500]
```

The effective temperature can usually be obtained from astronomical databases such as SIMBAD or from the literature for the target of interest.

### Other Generic Instrument Parameters

Apart from the parameters described above, the remaining Generic Instrument parameters should generally follow the recommendations provided in the official LBL documentation.

## 6.2 Required Header Keywords

LBL reads the observation metadata from the headers of the FITS files generated by the adapter. The corresponding `KW_*` parameters in the wrap file must therefore be configured before running LBL.

The generated wrap file includes a description and an example value for each header-keyword parameter. In most cases, the example values provided by LBL can be copied directly.

For example, the mid-exposure time should be read from the `MJDMID` keyword:

```python
rparams['KW_MID_EXP_TIME'] = "MJDMID"
```

### Signal-to-Noise Keywords

The science FITS files contain two separate signal-to-noise values:

```python
rparams['KW_SNR'] = "SNR"
rparams['KW_EXT_SNR'] = "EXT_SNR"
```

`SNR` stores the general signal-to-noise estimate, while `EXT_SNR` stores the signal-to-noise value measured in the selected reference order.

The use of `EXT_SNR` is the main difference from the example configuration provided in the generated wrap file. A separate keyword is used to avoid confusing the reference-order signal-to-noise value with the general `SNR` value.

For the remaining header-keyword parameters, follow the descriptions and recommended examples included in the generated LBL wrap file.


## 6.3 Order Selection

The order selection consists of two separate configuration steps:

1. Select the physical spectral orders to retain during data conversion.
2. Specify the corresponding wavelength range in the LBL wrap file.

These two configurations must always be consistent.

### Adapter Configuration

The adapter determines which physical spectral orders from the original reduced spectra are written into the science and blaze FITS files.

For example,

```python
LOWER_ORDER = 84
UPPER_ORDER = 114
```

retains only orders 84–114 in the converted LBL input files.

### Wrap File Configuration

After selecting the order range in the adapter, the corresponding wavelength range should be updated in the generated wrap file. These wavelength limits define the spectral region that LBL will analyze.

If the wavelength limits in the wrap file do not match the selected orders in the converted FITS files, LBL may attempt to access spectral orders that are not present in the data.

If a different order range is required, first update the adapter configuration, regenerate the converted FITS files, and then modify the wavelength range in the wrap file accordingly.

The correspondence between physical order number and wavelength for the Wendelstein/FOCES spectrograph can be found in **Appendix V of Hanna's thesis**, which may be used as a reference when selecting a different wavelength range.


## 6.4 Wrap File Checklist

Before running LBL, verify that the generated wrap file has been updated appropriately.

- [ ] `INSTRUMENT` is set to `"Generic"`.
- [ ] `OBJECT_COMPARISON` is set to the science target name.
- [ ] `OBJECT_TEFF` has been updated to the target's effective temperature.
- [ ] `KW_EXT_SNR` is set to `"EXT_SNR"`.
- [ ] All remaining `KW_*` parameters follow the recommendations provided in the generated LBL wrap file.
- [ ] The wavelength range in the wrap file matches the spectral orders selected in the adapter.

Once these settings have been verified, the converted FITS files are ready to be processed by LBL. After LBL has completed successfully, the resulting `.rdb` files can be analyzed using `lbl_analysis.py`, which is described in the next chapter.


# 7. Using lbl_analysis.py

After LBL has completed successfully, the primary outputs are stored as `.rdb` files. These files contain the radial velocity measurements and associated quantities produced by the LBL pipeline.

The `lbl_analysis.py` script provides a collection of visualization and analysis tools for inspecting these outputs. It is intended to facilitate exploratory analysis, data quality assessment, and comparison between different LBL runs.

The following sections describe the basic workflow and the main analysis functions available in the script.


## 7.1 Workflow

The typical workflow for `lbl_analysis.py` is

```text
LBL .rdb files
      │
      ▼
Load input files
      │
      ▼
Select analysis function
      │
      ▼
Generate figures
      │
      ▼
Display or save results
```

Generated figures can either be displayed interactively or saved to disk.


## 7.2 Analysis Functions

The script currently provides three analysis functions.

### Time-series Analysis

Generate radial velocity time-series plots to inspect the measurements over time and identify potential trends or outliers.

### Period Analysis

Generate Lomb–Scargle periodograms to search for periodic signals in the radial velocity measurements.

### Phase-folded Analysis

Fold the radial velocity measurements using a selected period and reference time to inspect the signal as a function of orbital phase.


# 8. Troubleshooting

This chapter summarizes several common issues that may occur when running LBL with the Wendelstein Generic Instrument configuration and their corresponding solutions.

## 8.1 PHOENIX Model Download Failure

### Problem

LBL fails while downloading the required PHOENIX stellar model.

### Solution

The PHOENIX models can be downloaded manually from the official archive:

https://phoenix.astro.physik.uni-goettingen.de/data/HiResFITS/PHOENIX-ACES-AGSS-COND-2011/

The required filename is usually shown in the LBL error message. Download the corresponding file from the archive above and copy it into the directory expected by LBL, keeping the original filename unchanged.

After the file has been placed in the correct directory, rerun the wrap file.

---

## 8.2 Missing `KW_MID_EXP_TIME` in `generic.py`

### Problem

Although the wrap file contains

```python
rparams['KW_MID_EXP_TIME'] = "MJDMID"
```

LBL still fails to recognize the middle-of-exposure time.

### Solution

Open the Generic Instrument implementation:

```text
lbl/instruments/generic.py
```

At approximately line 650, locate the function

```python
def populate_sci_table(self, filename: str, tdict: dict,
                       sci_hdr: io.LBLHeader, berv: float = 0.0) -> dict:
```

Inside this function, find the following list:

```python
drs_keys = ['KW_MJDATE', 'KW_EXPTIME', 'KW_DATE', 'KW_DPRTYPE',
            'KW_OBJNAME', 'KW_EXT_SNR']
```

Add `'KW_MID_EXP_TIME'` to the list:

```python
drs_keys = ['KW_MJDATE', 'KW_MID_EXP_TIME', 'KW_EXPTIME',
            'KW_DATE', 'KW_DPRTYPE', 'KW_OBJNAME', 'KW_EXT_SNR']
```

Save `generic.py` and rerun the wrap file.

> **Note:** This modification is made to the installed LBL package. If LBL is updated or reinstalled, the change may need to be applied again.

---

## 8.3 Missing `APPROX_RESOLUTION`

### Problem

LBL terminates during the template-building stage with an error similar to:

```text
TypeError: unsupported operand type(s) for /: 'float' and 'NoneType'
```

### Solution

This error is usually caused by a missing `APPROX_RESOLUTION` parameter in the wrap file.

Add the following lines to the instrument configuration section of the wrap file:

```python
# Approximate mean spectral resolving power, R = lambda / delta_lambda
rparams['APPROX_RESOLUTION'] = 60000
```

Make sure this parameter is defined before calling

```python
lbl_wrap(rparams)
```

Save the wrap file and rerun LBL.
